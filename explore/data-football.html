📊 데이터로 읽는 축구 — 전문가의 시선

"감독의 감(感)이 옳다"를 어떻게 증명할까? 현대 축구는 감을 버리지 않는다. 오히려 감을 숫자로 검증하고 보강한다. 이 페이지는 프로 구단·분석회사가 실제로 쓰는 지표와 사고방식을 — 어렵더라도 — 진짜로 다룬다.

오해부터 풀자. 데이터 분석은 "감 대 데이터"의 싸움이 아니다. 좋은 분석가는 코치의 직관을 가설로 보고, 데이터로 그 가설을 검증한다. "우리가 측면을 못 막는 것 같다"는 느낌 → "오른쪽 하프스페이스에서 내준 슛 xG가 리그 평균의 1.8배"라는 숫자로 바꾸는 일. 느낌을 측정 가능한 것으로 번역하는 게 핵심이다.

📌 큰 그림: 데이터 → 정보 → 의사결정. 경기장에서 일어난 사건과 위치를 기록(데이터)하고, 그것을 지표로 요약(정보)한 뒤, 영입·전술·훈련 같은 결정으로 잇는다. 이 세 단계를 따라가 보자.

1. 데이터의 두 종류 — 이벤트 데이터 vs 트래킹 데이터

축구 데이터는 크게 두 종류다. 이 구분을 모르면 어떤 지표가 왜 가능한지(또는 불가능한지) 이해할 수 없다.

💬 이벤트 데이터(event data)란? 경기 중 일어난 '사건'을 하나씩 기록한 표다. "37분 12초, 손흥민, 패스, 시작좌표→끝좌표, 성공". 패스·슛·태클·드리블·파울 같은 온더볼(on-the-ball, 공을 가진 순간) 행동이 중심이야. Opta(옵타)·StatsBomb(스탯봄) 같은 회사가 사람+AI로 경기당 수천 건씩 태깅한다.
💬 트래킹 데이터(tracking data)란? 카메라·센서로 모든 선수 22명 + 공의 (x, y) 좌표를 초당 25번 찍은 위치 스트림이야. 한 경기면 수백만 개의 좌표점이 쌓이지. 공이 없는 선수의 오프더볼(off-the-ball) 움직임까지 볼 수 있는 게 결정적 차이다.
구분이벤트 데이터트래킹 데이터
기록 대상'사건'(패스·슛·태클…) + 그 좌표22명 + 공의 연속 위치(x,y)
해상도사건이 일어날 때마다(경기당 수천 건)시간 연속, 보통 25fps(경기당 수백만 점)
공 없는 선수거의 안 보임완전히 보임(수비 라인·간격·침투)
대표 출처Opta, StatsBomb광학 트래킹(다중 카메라), 일부 GPS 조끼
주로 답하는 질문"무엇을, 어디서 했나?""공간을 어떻게 점유·통제했나?"
데이터 크기작다(표 한 장 느낌)매우 크다(영상급)
💡 둘은 경쟁이 아니라 보완이다. xG(다음 절) 같은 기본 지표는 이벤트 데이터만으로도 만들지만, "패스를 받을 공간이 정말 비어 있었나"처럼 공간을 따지는 고급 지표는 트래킹 데이터가 있어야 한다. 그래서 현장에서는 둘을 합쳐서(merge) 쓴다.

2. 기본 → 고급 지표 사다리

지표에는 층(layer)이 있다. 아래로 갈수록 "겉보기"에서 "본질"로, "결과"에서 "과정의 질"로 내려간다. 위 칸은 누구나 아는 것, 아래 칸은 분석가가 보는 것이다.

  1. 슈팅·유효슈팅·점유율 — 가장 기초. 그러나 함정도 크다. 점유율의 함정?: 공을 많이 가졌다고 잘한 게 아니다. 어디서 가졌느냐가 중요하다.
  2. 패스성공률 · 전진패스(progressive pass) — 전진패스?는 "안전하게 돌린 패스"와 "위험을 무릅쓰고 앞으로 찌른 패스"를 나눈다. 성공률 95%여도 전부 옆패스면 위협이 없다.
  3. 기대어시스트 xA(expected Assists) — xA?는 찬스를 만든 패스의 질을 잰다. 동료가 헛발질해도 그 패스가 좋았다면 xA로 인정받는다.
  4. 압박 지표 PPDA — PPDA?(상대 패스 ÷ 우리 수비행동). 숫자가 낮을수록 상대에게 패스 몇 번 못 하게 강하게 압박했다는 뜻. 게겐프레싱 팀이 PPDA가 낮다.
  5. 필드 틸트(field tilt) — 필드 틸트?: 전체 점유율 대신 위협적인 위치(상대 진영 깊은 곳)에서의 점유 비율. "영역의 기울기"라고 보면 된다. 점유율은 50:50이어도 필드 틸트가 70:30이면 사실상 한 팀이 상대를 가둔 경기다.
  6. 세트피스 가치 — 코너·프리킥·스로인 같은 정지 상황은 통제 가능한 변수라 따로 모델링한다. 상위 리그 득점의 약 1/4~1/3이 세트피스에서 나온다고 대체로 알려져, 전담 코치를 둘 만큼 중요하다.
📌 사다리를 관통하는 원칙: "결과(골)는 적고 운이 많이 섞인다. 그래서 결과 대신 '결과로 이어질 과정의 질'을 측정한다." 이것이 다음 절 xG의 핵심 철학이다.

3. xG(기대득점) 심층 — 현대 축구 분석의 심장

하나만 제대로 이해해야 한다면 xG다. 거의 모든 고급 지표가 xG의 사고방식을 빌려 쓴다.

💬 xG(expected Goals, 기대득점)란? 슛 하나하나가 골이 될 확률(0~1)을 모두 더한 값이야. "이 위치·각도의 슛은 역사적으로 100번 중 23번 들어갔다"면 그 슛의 xG = 0.23. 한 경기 모든 슛의 xG를 더하면 "이 팀이 만든 찬스의 질로 보면 평균 몇 골이 났어야 하는가"가 나온다.

수식으로는 이렇게 쓴다(겁먹지 말 것 — "확률을 다 더한다"는 뜻일 뿐):

xG(경기) = Σ  P(골 | 슛 위치·각도·상황)
          모든 슛

# 예: 슛 3개의 골 확률이 0.35, 0.05, 0.12 라면
# xG = 0.35 + 0.05 + 0.12 = 0.52  (→ "평균 0.52골짜리 찬스를 만들었다")

무엇으로 계산하나? 과거 수십만 개의 슛을 학습해, 새 슛의 골 확률을 추정한다. 주로 쓰는 입력(특징, feature):

  • 슛 거리 — 골문에서 가까울수록 확률 ↑ (가장 강력한 변수)
  • 슛 각도 — 골문을 얼마나 넓게 볼 수 있나(정면일수록 ↑, 골라인 부근 좁은 각 ↓)
  • 신체 부위 — 발 vs 머리(헤더는 보통 확률 ↓)
  • 어시스트 종류 — 스루패스·크로스·컷백 등 직전 패스의 성격
  • 상황 — 오픈플레이 / 세트피스 / 역습 / 패널티(패널티는 약 0.76으로 거의 고정)
  • 원터치 여부 — 정리할 시간이 있었나(원터치 발리는 난도 ↑)
  • 수비 압박 — 슛 순간 앞을 막은 수비수 수(트래킹 데이터가 있으면 추가)

예를 들어, 같은 "한 번의 슛"이라도 위치에 따라 xG가 이렇게 다르다(설명용 예시 수치):

0.76패널티 0.35박스 중앙 0.12박스 외곽 0.05측면 좁은 각 0.03중거리 25m 0.02하프라인 근처

숫자는 설명용 예시 xG. 같은 슛이라도 박스 중앙(0.35)과 측면 좁은 각(0.05)은 약 7배 차이. "왜 좋은 위치를 만들어야 하는가"가 숫자로 보인다.

xG로 무엇을 아나?

  • 운 vs 실력 분리 — 적은 찬스로 운 좋게 이겼는지, 압도하고도 못 넣었는지 구분한다.
  • 과대/과소 성과 판별 — 실제 득점 > xG가 오래 지속되면 "비현실적으로 운이 좋았다 → 곧 평균으로 회귀(regression to the mean)"라고 본다.
  • 선수 평가 — 스트라이커가 xG보다 꾸준히 더 많이 넣으면 '결정력(finishing)' 신호일 수 있다(샘플이 클 때만).
💬 파생 지표 한 묶음: npxG = non-penalty xG, 패널티(0.76으로 거의 고정이라 실력 신호가 약함)를 뺀 xG. xGA = 상대에게 내준 슛들의 xG(수비의 질, 낮을수록 좋음). xGD = xG − xGA, "찬스의 질"로 본 우리 팀의 순(純) 우위.
⚠️ 가장 흔한 오해: xG는 '예언'이 아니다. "xG 2.0이면 2골 넣는다"가 아니라, "이 정도 질의 찬스를 아주 여러 번 반복하면 평균적으로 약 2골이 난다"는 뜻이다. 축구는 한 경기 골이 워낙 적어서(평균 2~3골) 단일 경기의 xG와 실제 득점은 크게 어긋날 수 있다. xG는 '운이 빠진, 슛 질의 누적'이지 점쟁이가 아니다. 표본이 커질수록(여러 경기) 실제와 가까워진다.

실제 득점 vs xG — "운"과 "실력"을 가르는 그래프 (가상의 세 팀)

막대그래프는 인터넷 연결 시 표시됩니다. 숫자(가상 예시) — A팀: 실제 22골 / xG 14.5 → +7.5(과대성과: 운·결정력으로 더 넣음, 회귀 위험). B팀: 실제 10골 / xG 16.8 → −6.8(과소성과: 찬스는 좋은데 못 넣음, 반등 기대). C팀: 실제 15골 / xG 15.2 → 거의 일치(질만큼 득점).

4. 슛맵 · 패스 네트워크 — 좌표를 그림으로

이벤트·트래킹 데이터의 핵심 무기는 모든 사건에 (x, y) 좌표가 붙어 있다는 점이다. 좌표가 있으면 표가 아니라 그림으로 볼 수 있고, 사람의 눈은 그림에서 패턴을 훨씬 빨리 잡는다.

🎯 슛맵(shot map)

경기장 위에 모든 슛을 점으로 찍는다. 점의 위치=슛 위치, 점의 크기=xG, 색=골/노골. 한 장만 봐도 "이 팀은 박스 안에서 좋은 슛을 만들었나, 멀리서 무리한 슛만 했나"가 보인다.

🕸️ 패스 네트워크(pass network)

선수를 평균 위치에 점으로 놓고, 둘 사이 패스가 많을수록 굵은 선으로 잇는다. 점의 위치=그 선수가 주로 머문 곳, 선의 굵기=연결 강도. 팀의 실제 형태와 중심축(누구를 거쳐 공이 도는지)이 드러난다.

시각화점/노드 = ?크기 = ?선/색 = ?한눈에 보는 것
슛맵슛 위치xG(찬스 질)색=골/노골찬스의 질과 위치
패스 네트워크선수 평균 위치패스 관여량선 굵기=연결 강도팀의 실제 형태·중심축
히트맵(heatmap)—(영역 밀도)—색 진하기=체류 시간선수·팀이 주로 머문 영역
💡 같은 90분이라도 "표"로 보면 슛 14개·점유율 58%일 뿐이지만, 슛맵으로 보면 "14개 중 12개가 박스 밖 무리한 슛"이라는 진짜 이야기가 보인다. 시각화는 데이터를 '읽는 속도'를 바꾼다.

5. 분석 문화 — '머니볼'은 축구에서 어떻게 작동했나

데이터를 잘 쓰면 돈을 적게 쓰고도 효율을 올릴 수 있다. 야구의 '머니볼'을 축구에 옮긴 사례로 대체로 거론되는 팀들이 있다. (단정이 아니라 "널리 알려진 경향"으로 읽자.)

🐝 브렌트포드

데이터 기반 영입·세트피스 최적화로 알려진 구단. 저예산으로 상위 리그에 안착했다고 평가받는다.

🕊️ 브라이튼

저평가 선수를 데이터로 발굴해 키운 뒤 비싸게 파는 모델로 대체로 언급된다.

🔴 리버풀

물리학·데이터 출신을 영입부서·코칭에 둔 것으로 알려진 대표적 빅클럽 사례.

⚠️ 주의: "데이터를 썼더니 무조건 성공"은 아니다. 같은 데이터를 쓰고도 결과가 갈린다. 데이터는 좋은 질문을 던지는 사람과 현장의 실행이 있을 때 비로소 힘을 낸다. 만능 열쇠가 아니다.
🔗 우리 앱과 연결: 거창해 보이지만, 우리 월드컵 앱의 js/standings.js도 같은 일을 한다 — 경기 결과(데이터)를 받아 승점·골득실 규칙으로 계산해 순위(정보)로 바꾼다. "원자료 → 규칙으로 요약 → 결정에 쓰는 정보"라는 데이터 분석의 뼈대를, 축소판으로 이미 만들어 본 셈이다. 프로의 xG 모델은 이 calcStandings()의 규칙이 훨씬 정교해진 것일 뿐이다.

6. 미션 — 직접 '데이터 → 정보'를 만들어 보자

초급

우리 앱에 팀별 '평균 득점/실점' 지표 추가하기. standings.js는 이미 팀마다 GF(득점)·GA(실점)·P(경기 수)를 쌓는다. 여기에 "경기당 평균 득점 = GF ÷ P", "경기당 평균 실점 = GA ÷ P"를 더해 순위표에 칸을 하나 늘려 보자.

avgFor = (team.P > 0) ? team.GF / team.P : 0; // 경기당 평균 득점 (0으로 나누기 방지) avgAga = (team.P > 0) ? team.GA / team.P : 0; // 경기당 평균 실점

🤖 Claude에게 이렇게: "standings.js의 calcStandings가 돌려주는 팀 객체에 경기당 평균 득점(avgFor)·평균 실점(avgGa) 필드를 추가하고, P가 0일 때 0으로 나누지 않게 처리해줘. 그리고 app.js 순위표에 칸 두 개를 추가해줘."

중급

한 경기를 보며 '나만의 xG'를 흉내 내기. 종이에 경기장을 그리고, 슛이 나올 때마다 위치에 점을 찍은 뒤 위 3절의 위치별 예시 xG 표(0.76 / 0.35 / 0.12 / 0.05 / 0.03 / 0.02)를 슛에 하나씩 갖다 붙여 더해 보자. 90분 뒤 "내 xG 합"과 "실제 골"을 비교하면, 그 경기가 운인지 실력인지 네가 직접 판정한 것이다.

🤖 Claude에게 이렇게: "내가 슛 위치(거리·각도)를 입력하면 아주 단순한 규칙(거리·각도→확률)으로 대략적인 xG를 돌려주는 작은 자바스크립트 함수를 만들어줘. 진짜 모델은 아니고 학습용으로, 왜 그렇게 계산하는지 주석도 달아줘."

더 깊이 — xG 모델을 실제로 어떻게 만드는지(지도학습), 공간을 점수로 바꾸는 xT·VAEP, 트래킹 파이프라인, 그리고 아빠의 NPU/반도체 세계와 만나는 지점은 심화편에서 다룬다.
